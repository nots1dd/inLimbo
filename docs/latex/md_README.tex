The in\+Limbo project aims to be a new upcoming TUI music player for UNIX based systems that gives music lovers a clean and efficient environment to browse, play and interact with your favourite offline music.\hypertarget{md_README_autotoc_md20}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{md_README_autotoc_md20}

\begin{DoxyItemize}
\item {\bfseries{Fully configurable}}\+: \doxylink{structKeybinds}{Keybinds}, library, FTP credentials, and more, all in one {\ttfamily config.\+toml} file located in {\ttfamily \$HOME/.config/in\+Limbo/config.\+toml}
\item {\bfseries{Very fast and easy to use}}\+: Optimized for speed and usability.
\item {\bfseries{Very few external dependencies}}\+: Minimal setup required to get started.
\item {\bfseries{Clean and nice TUI experience}}\+: Interact with your music library seamlessly in a terminal interface.
\item {\bfseries{Showcases all the metadata}}\+: Displays song metadata, including lyrics, genre, and more, for every file in your library.
\item {\bfseries{Plays and caters to many audio file formats}}\+: Compatible with a variety of audio formats, depending on the capabilities of \href{https://github.com/mackron/miniaudio}{\texttt{ miniaudio}}.
\item {\bfseries{Customizable}}\+: Add your own color palette to in\+Limbo ;)
\end{DoxyItemize}\hypertarget{md_README_autotoc_md21}{}\doxysubsection{\texorpdfstring{LOOKS}{LOOKS}}\label{md_README_autotoc_md21}
   \hypertarget{md_README_autotoc_md22}{}\doxysubsection{\texorpdfstring{DEPENDENCIES}{DEPENDENCIES}}\label{md_README_autotoc_md22}

The goal of this project is just to find peace and solitude for me as I try to create something I wanted for a long time

Here are a few novelties that may or may not have been added yet\+:


\begin{DoxyEnumerate}
\item Cool and amazing TUI experience thanks to FTXUI
\item Full use of metadata (lyrics, genre, thumbnail, etc..) and available for display
\item Simple and easy network transfer of your favourite songs over to another device
\item Fast and efficient loading of your song directory with everything you need \+:)
\item TUI over the web?? (Why would you want this)
\item Active testing of the codebase using GTest Suite
\item Security ensured using Code\+QL workflow
\end{DoxyEnumerate}\hypertarget{md_README_autotoc_md24}{}\doxysubsection{\texorpdfstring{{\bfseries{BUILDING}}}{{\bfseries{BUILDING}}}}\label{md_README_autotoc_md24}

The in\+Limbo project is aiming to be more aware of how bad its codebase is, hence I am setting up unit tests for most if not all functions/classes that should\+:


\begin{DoxyEnumerate}
\item Give more clarity on how the logical flow of the function/class works in a test suite
\item Logical flaws and boundary conditions going unchecked can be understood better
\item Should give more confidence in the code structure if such tests pass
\end{DoxyEnumerate}
