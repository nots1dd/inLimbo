The in\+Limbo project aims to be a new upcoming TUI music player for UNIX based systems that gives music lovers a clean and efficient environment to browse, play and interact with your favourite offline music.\hypertarget{md_README_autotoc_md20}{}\doxysubsection{\texorpdfstring{{\bfseries{Why}}}{{\bfseries{Why}}}}\label{md_README_autotoc_md20}
There are a dozen of great music players out there that do a really good job of making my music experience top notch, but nothing was ever perfect. So I tried making something that in my opinion comes closest to my ideal TUI music player.

Why would you want to stay in Limbo?

Sometimes it feels good to be stuck in a weird place in your life, and it feels even better to be stuck in Limbo with music \+:)\hypertarget{md_README_autotoc_md21}{}\doxysubsection{\texorpdfstring{{\bfseries{Features}}}{{\bfseries{Features}}}}\label{md_README_autotoc_md21}

\begin{DoxyItemize}
\item {\bfseries{Fully configurable}}\+: \doxylink{structKeybinds}{Keybinds}, library, FTP credentials, and more, all in one {\ttfamily config.\+toml} file located in {\ttfamily \$HOME/.config/in\+Limbo/config.\+toml}
\item {\bfseries{Very fast and easy to use}}\+: Optimized for speed and usability.
\item {\bfseries{Very few external dependencies}}\+: Minimal setup required to get started.
\item {\bfseries{Clean and nice TUI experience}}\+: Interact with your music library seamlessly in a terminal interface.
\item {\bfseries{Showcases all the metadata}}\+: Displays song metadata, including lyrics, genre, and more, for every file in your library.
\item {\bfseries{Plays and caters to many audio file formats}}\+: Compatible with a variety of audio formats, depending on the capabilities of \href{https://github.com/mackron/miniaudio}{\texttt{ miniaudio}}.
\item {\bfseries{Customizable}}\+: Heavily customizable TUI with custom keybinds, colors, UI designs and much more to come...
\item {\bfseries{Privacy}}\+: No data is collected by in\+Limbo other than a backtrace of when the application catches a critical signal like {\ttfamily SIGSEGV}, {\ttfamily SIGABRT}
\end{DoxyItemize}\hypertarget{md_README_autotoc_md22}{}\doxysubsection{\texorpdfstring{{\bfseries{LOOKS}}}{{\bfseries{LOOKS}}}}\label{md_README_autotoc_md22}

\hypertarget{md_README_autotoc_md28}{}\doxysubsection{\texorpdfstring{Cache and Directory Management}{Cache and Directory Management}}\label{md_README_autotoc_md28}
If you update the song directory in {\ttfamily config.\+toml}, {\bfseries{in\+Limbo}} does not automatically remove the static serialized {\ttfamily lib.\+bin} file, which is used to load the song map.

To ensure that {\bfseries{in\+Limbo}} loads the directory dynamically, use the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{inLimbo\ -\/-\/update-\/cache-\/run}

\end{DoxyCode}


You can also merge the dynamic load with other arguments like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{inLimbo\ -\/-\/update-\/cache-\/run\ -\/-\/print-\/song-\/tree\ \#\ will\ create\ a\ new\ song\ map\ and\ print\ it\ regardless\ of\ lib.bin\ data}

\end{DoxyCode}


Note that this will result in slower song map times as the song map is generated dynamically on each run.\hypertarget{md_README_autotoc_md29}{}\doxysubsection{\texorpdfstring{{\bfseries{Command-\/\+Line Arguments}}}{{\bfseries{Command-\/\+Line Arguments}}}}\label{md_README_autotoc_md29}

\begin{DoxyItemize}
\item {\ttfamily -\/-\/help} {\bfseries{Description}}\+: Display help information about available arguments for in\+Limbo.
\item {\ttfamily -\/-\/version} {\bfseries{Description}}\+: Display the version number of in\+Limbo.
\item {\ttfamily -\/-\/clear-\/cache} {\bfseries{Description}}\+: Clear cached data, which includes the song map and other temporary data stored by the application.
\item {\ttfamily -\/-\/show-\/config-\/file} {\bfseries{Description}}\+: Display the path to the configuration file used by in\+Limbo.
\item {\ttfamily -\/-\/show-\/log-\/dir} {\bfseries{Description}}\+: Show the directory path where logs are stored for in\+Limbo.
\item {\ttfamily -\/-\/show-\/dbus-\/name} {\bfseries{Description}}\+: Show the DBus service name used by in\+Limbo.
\item {\ttfamily -\/-\/update-\/cache-\/run} {\bfseries{Description}}\+: Update the cache file and run the application. This will make in\+Limbo load the directory dynamically every time (note that this will result in slower song map times).
\item {\ttfamily -\/-\/print-\/song-\/tree} {\bfseries{Description}}\+: Print the song map parsed from the directory. This shows the structure of the song tree and how songs are organized.
\item {\ttfamily -\/-\/print-\/artists-\/all} {\bfseries{Description}}\+: Print all parsed artists from the song map. This will list all artists found in the directory and their corresponding songs.
\item {\ttfamily -\/-\/print-\/songs-\/by-\/artist} {\bfseries{Description}}\+: Print all the songs of a given artist. This argument requires you to specify the artist’s name, and it will list all songs by that artist.
\item {\ttfamily -\/-\/print-\/songs-\/by-\/genre-\/all} {\bfseries{Description}}\+: Print all parsed genres and their song mappings. This will show all the genres available in the song map and list songs under each genre.
\item {\ttfamily -\/-\/print-\/song-\/info} {\bfseries{Description}}\+: Print every parsed information (even default info) regarding a song (Arguments\+: Can take {\bfseries{SONG NAME}} or {\bfseries{FILE PATH}})
\end{DoxyItemize}

-\/\texorpdfstring{$>$}{>} Undefined Behaviour Sanitation (UBSan)\hypertarget{md_README_autotoc_md39}{}\doxysubsection{\texorpdfstring{{\bfseries{DIRECTORY SORTING}}}{{\bfseries{DIRECTORY SORTING}}}}\label{md_README_autotoc_md39}
in\+Limbo reads a directory and attempts to parse each inode for its metadata using Tag\+Lib, and accordingly creates a song map datastructure that is utilized to efficiently traverse the required artist, album, song etc.

To know more about in\+Limbo Project\textquotesingle{}s logical flow on creating a song map, visit \href{https://github.com/nots1dd/inLimbo/blob/main/src/dirsort/SONGMAP.md}{\texttt{ SONGMAP}}\hypertarget{md_README_autotoc_md40}{}\doxysubsection{\texorpdfstring{{\bfseries{TESTING}}}{{\bfseries{TESTING}}}}\label{md_README_autotoc_md40}
The in\+Limbo project is aiming to be more aware of how bad its codebase is, hence I am setting up unit tests for most if not all functions/classes that should\+:


\begin{DoxyEnumerate}
\item Give more clarity on how the logical flow of the function/class works in a test suite
\item Logical flaws and boundary conditions going unchecked can be understood better
\item Should give more confidence in the code structure if such tests pass
\end{DoxyEnumerate}

The testing is done using GTest, read \href{https;//github.com/nots1dd/inLimbo/blob/main/tests/TESTS,md}{\texttt{ TESTS.\+md}} to understand more on how tests are configured and built\hypertarget{md_README_autotoc_md41}{}\doxysubsection{\texorpdfstring{{\bfseries{DOCUMENTATION}}}{{\bfseries{DOCUMENTATION}}}}\label{md_README_autotoc_md41}
