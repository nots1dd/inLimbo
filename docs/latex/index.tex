\hypertarget{index_modules}{}\doxysection{\texorpdfstring{List of modules.}{List of modules.}}\label{index_modules}
The project is comprised of 3 modules\+:


\begin{DoxyEnumerate}
\item {\bfseries{ftxui/screen}} defines a {\ttfamily \doxylink{classftxui_1_1Screen}{ftxui\+::\+Screen}}, a grid of {\ttfamily \doxylink{structftxui_1_1Pixel}{ftxui\+::\+Pixel}}.
\item {\bfseries{ftxui/dom}} is the main module. It defines a hierarchical set of {\ttfamily ftxui\+::\+Element}. An element draws something on the {\ttfamily \doxylink{classftxui_1_1Screen}{ftxui\+::\+Screen}}. It is responsive to the size of its container.
\item {\bfseries{ftxui/component}} The module is required if your program needs to respond to user input. It defines a set of {\ttfamily ftxui\+::\+Component}. These components can be utilized to navigate using the arrow keys {\itshape and/or} cursor. There are several builtin widgets like checkbox/inputbox/etc to interact with. You can combine them, or even define your own custom components.
\end{DoxyEnumerate}\hypertarget{index_module-screen}{}\doxysection{\texorpdfstring{screen}{screen}}\label{index_module-screen}
This is the visual element of the program. It defines a {\ttfamily \doxylink{classftxui_1_1Screen}{ftxui\+::\+Screen}}, which is a grid of {\ttfamily \doxylink{structftxui_1_1Pixel}{ftxui\+::\+Pixel}}. A Pixel represents a Unicode character and its associated style (bold, colors, etc.). The screen can be printed as a string using {\ttfamily \doxylink{classftxui_1_1Screen_a99aaed52447e5693a3819c382fc69abf}{ftxui\+::\+Screen\+::\+To\+String()}}.  \hypertarget{index_module-component}{}\doxysection{\texorpdfstring{component}{component}}\label{index_module-component}
The {\ttfamily ftxui\+::component} module defines the logic that produces interactive components that respond to user events (keyboard, mouse, etc.).

A {\ttfamily \doxylink{classftxui_1_1ScreenInteractive}{ftxui\+::\+Screen\+Interactive}} defines a main loop that renders a component.

A {\ttfamily ftxui\+::\+Component} is a shared pointer to a {\ttfamily \doxylink{classftxui_1_1ComponentBase}{ftxui\+::\+Component\+Base}}. The latter defines\+:
\begin{DoxyItemize}
\item {\ttfamily \doxylink{classftxui_1_1ComponentBase_ab27effc8e19a21b2b06c030629e62ee5}{ftxui\+::\+Component\+Base\+::\+Render()}}\+: How to render the interface.
\item {\ttfamily \doxylink{classftxui_1_1ComponentBase_a54ed139466bbcd324f27ceecb20751c7}{ftxui\+::\+Component\+Base\+::\+On\+Event()}}\+: How to react to events.
\item {\ttfamily \doxylink{classftxui_1_1ComponentBase_a206236d7672399c26f0d8309538e4cab}{ftxui\+::\+Component\+Base\+::\+Add()}}\+: Construct a parent/child relationship between two components. The tree of component is used to define how to navigate using the keyboard.
\end{DoxyItemize}

{\ttfamily ftxui\+::\+Element} are used to render a single frame.

{\ttfamily ftxui\+::\+Component} are used to render dynamic user interface, producing multiple frame, and updating its state on events.

\href{https://arthursonzogni.github.io/FTXUI/examples_2component_2gallery_8cpp-example.html}{\texttt{ Gallery}} of multiple components. (\href{https://arthursonzogni.github.io/FTXUI/examples/?file=component/gallery}{\texttt{ demo}})



All predefined components are available in \href{./component_8hpp.html}{\texttt{ "{}ftxui/dom/component.\+hpp"{}}}

\hypertarget{index_component-force-redraw}{}\doxysubsection{\texorpdfstring{Force a frame redraw.}{Force a frame redraw.}}\label{index_component-force-redraw}
Typically, {\ttfamily ftxui\+::\+Screen\+Interactive\+::\+Loop()} is responsible for drawing a new frame whenever a new group of events (e.\+g keyboard, mouse, window resize, etc.) has been processed. However, you might want to react to arbitrary events that are unknown to FTXUI. To accomplish this, you must post events using {\ttfamily \doxylink{classftxui_1_1ScreenInteractive_a0cb88888962608ed07724350a97d7cea}{ftxui\+::\+Screen\+Interactive\+::\+Post\+Event}} ({\bfseries{this is thread safe}}) via a thread. You will have to post the event {\ttfamily ftxui\+::\+Event\+::\+Custom}.

Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{screen-\/>PostEvent(Event::Custom);}

\end{DoxyCode}
