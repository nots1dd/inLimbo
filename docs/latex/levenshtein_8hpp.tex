Computes the Levenshtein distance between two strings. 

The Levenshtein distance is calculated by finding the minimum number of operations required to transform one string into another. The allowed operations are\+:
\begin{DoxyItemize}
\item {\bfseries{Insertion}}\+: Insert a character into one string.
\item {\bfseries{Deletion}}\+: Delete a character from one string.
\item {\bfseries{Substitution}}\+: Replace a character in one string with another.
\end{DoxyItemize}

This implementation uses dynamic programming to compute the Levenshtein distance. A 2D table is constructed, where each entry at \mbox{[}i\mbox{]}\mbox{[}j\mbox{]} contains the distance between the first i characters of s1 and the first j characters of s2. The final distance is found at the bottom-\/right corner of the table. The time complexity of this algorithm is O(m \texorpdfstring{$\ast$}{*} n), where m and n are the lengths of the two input strings. \texorpdfstring{$<$}{<} Length of the first string.
