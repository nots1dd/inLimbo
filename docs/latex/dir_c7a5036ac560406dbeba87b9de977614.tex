{\itshape Functional Terminal (X) User interface}

A simple cross-\/platform C++ library for terminal based user interfaces!\hypertarget{README.md_autotoc_md239}{}\doxysubsubsection{\texorpdfstring{Feature}{Feature}}\label{README.md_autotoc_md239}

\begin{DoxyItemize}
\item Functional style. Inspired by \href{https://hackernoon.com/building-reactive-terminal-interfaces-in-c-d392ce34e649?gi=d9fb9ce35901}{\texttt{ \mbox{[}1\mbox{]}}} and \href{https://reactjs.org/}{\texttt{ React}}
\item Simple and elegant syntax (in my opinion)
\item Keyboard \& mouse navigation.
\item Support for \href{https://en.wikipedia.org/wiki/UTF-8}{\texttt{ UTF8}} and \href{https://en.wikipedia.org/wiki/Halfwidth_and_fullwidth_forms}{\texttt{ fullwidth chars}} (→ 测试)
\item Support for animations. \href{https://arthursonzogni.github.io/FTXUI/examples/?file=component/menu_underline_animated_gallery}{\texttt{ Demo 1}}, \href{https://arthursonzogni.github.io/FTXUI/examples/?file=component/button_style}{\texttt{ Demo 2}}
\item Support for drawing. \href{https://arthursonzogni.github.io/FTXUI/examples/?file=component/canvas_animated}{\texttt{ Demo}}
\item No dependencies
\item {\bfseries{Cross platform}}\+: Linux/\+Mac\+OS (main target), Web\+Assembly, Windows (Thanks to contributors!).
\item Learn by examples, and tutorials
\item Multiple packages\+: CMake \href{[https://bewagner.net/programming/2020/05/02/cmake-fetchcontent/](https://cmake.org/cmake/help/latest/module/FetchContent.html)}{\texttt{ Fetch\+Content}} (preferred), vcpkg, pkgbuild, conan.
\item Good practises\+: documentation, tests, fuzzers, performance tests, automated CI, automated packaging, etc...
\end{DoxyItemize}\hypertarget{README.md_autotoc_md240}{}\doxysubsubsection{\texorpdfstring{Documentation}{Documentation}}\label{README.md_autotoc_md240}

\hypertarget{README.md_autotoc_md243}{}\doxysubsubsubsubsection{\texorpdfstring{DOM}{DOM}}\label{README.md_autotoc_md243}
This module defines a hierarchical set of Element. An element manages layout and can be responsive to the terminal dimensions.

They are declared in \href{https://arthursonzogni.github.io/FTXUI/elements_8hpp_source.html}{\texttt{ \texorpdfstring{$<$}{<}ftxui/dom/elements.\+hpp\texorpdfstring{$>$}{>}}}



{\bfseries{Layout}}\begin{adjustwidth}{1em}{0em}


Element can be arranged together\+:
\begin{DoxyItemize}
\item horizontally with {\ttfamily hbox}
\item vertically with {\ttfamily vbox}
\item inside a grid with {\ttfamily gridbox}
\item wrap along one direction using the {\ttfamily flexbox}.
\end{DoxyItemize}

Element can become flexible using the the {\ttfamily flex} decorator.

\href{https://arthursonzogni.github.io/FTXUI/examples_2dom_2vbox_hbox_8cpp-example.html}{\texttt{ Example}} using {\ttfamily hbox}, {\ttfamily vbox} and {\ttfamily filler}.



\href{https://arthursonzogni.github.io/FTXUI/examples_2dom_2gridbox_8cpp-example.html}{\texttt{ Example}} using gridbox\+:



\href{https://github.com/ArthurSonzogni/FTXUI/blob/master/examples/dom/hflow.cpp}{\texttt{ Example}} using flexbox\+:



\href{https://arthursonzogni.github.io/FTXUI/examples_2dom_2hflow_8cpp-example.html}{\texttt{ See}} also this \href{https://arthursonzogni.github.io/FTXUI/examples/?file=component/flexbox}{\texttt{ demo}}.

\end{adjustwidth}

The ftxui/component is needed when you want to produce dynamic UI, reactive to the user\textquotesingle{}s input. It defines a set of ftxui\+::\+Component. A component reacts to Events (keyboard, mouse, resize, ...) and Render Element (see previous section).

Prebuilt components are declared in \href{https://arthursonzogni.github.io/FTXUI/component_8hpp_source.html}{\texttt{ \texorpdfstring{$<$}{<}ftxui/component/component.\+hpp\texorpdfstring{$>$}{>}}}
