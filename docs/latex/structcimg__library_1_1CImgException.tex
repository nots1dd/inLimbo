\doxysubsection{Detailed Description}
Instances of {\ttfamily \doxylink{structcimg__library_1_1CImgException}{CImg\+Exception}} are thrown when errors are encountered in a \textbackslash{}\+CImg function call. 

\begin{DoxyParagraph}{Overview}

\end{DoxyParagraph}
\doxylink{structcimg__library_1_1CImgException}{CImg\+Exception} is the base class of all exceptions thrown by \textbackslash{}\+CImg (except {\bfseries{\doxylink{structcimg__library_1_1CImgAbortException}{CImg\+Abort\+Exception}}}). \doxylink{structcimg__library_1_1CImgException}{CImg\+Exception} is never thrown itself. Derived classes that specify the type of errord are thrown instead. These classes can be\+:


\begin{DoxyItemize}
\item {\bfseries{\doxylink{structcimg__library_1_1CImgAbortException}{CImg\+Abort\+Exception}\+:}} Thrown when a computationally-\/intensive function is aborted by an external signal. This is the only {\ttfamily non-\/derived} exception class.
\item {\bfseries{\doxylink{structcimg__library_1_1CImgArgumentException}{CImg\+Argument\+Exception}\+:}} Thrown when one argument of a called \textbackslash{}\+CImg function is invalid. This is probably one of the most thrown exception by \textbackslash{}\+CImg. For instance, the following example throws a {\ttfamily \doxylink{structcimg__library_1_1CImgArgumentException}{CImg\+Argument\+Exception}\+:} 
\begin{DoxyCode}{0}

\DoxyCodeLine{\mbox{\hyperlink{namespacecimg__library_aff6e86f25b2615655070e2de4913e63e}{img}}.mirror(\textcolor{charliteral}{'e'});\ \ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ Try\ to\ mirror\ image\ along\ the\ (non-\/existing)\ 'e'-\/axis}}

\end{DoxyCode}

\item {\bfseries{\doxylink{structcimg__library_1_1CImgDisplayException}{CImg\+Display\+Exception}\+:}} Thrown when something went wrong during the display of images in CImg\+Display instances.
\item {\bfseries{\doxylink{structcimg__library_1_1CImgInstanceException}{CImg\+Instance\+Exception}\+:}} Thrown when an instance associated to a called \textbackslash{}\+CImg method does not fit the function requirements. For instance, the following example throws a {\ttfamily \doxylink{structcimg__library_1_1CImgInstanceException}{CImg\+Instance\+Exception}\+:}

\DoxyCodeLine{\textcolor{keyword}{const}\ \textcolor{keywordtype}{float}\ value\ =\ \mbox{\hyperlink{namespacecimg__library_aff6e86f25b2615655070e2de4913e63e}{img}}.at(0);\ \ \ \textcolor{comment}{//\ Try\ to\ read\ first\ pixel\ value\ (does\ not\ exist)}}

\end{DoxyCode}

\item {\bfseries{\doxylink{structcimg__library_1_1CImgIOException}{CImg\+IOException}\+:}} Thrown when an error occurred when trying to load or save image files. This happens when trying to read files that do not exist or with invalid formats. \end{DoxyCode}

\item {\bfseries{\doxylink{structcimg__library_1_1CImgWarningException}{CImg\+Warning\+Exception}\+:}} Thrown only if configuration macro {\ttfamily cimg\+\_\+strict\+\_\+warnings} is set, and when a \textbackslash{}\+CImg function has to display a warning message (see \doxylink{namespacecimg__library_1_1cimg_a5ee7f021ed42b0ac32e1254414c3faab}{cimg\+::warn()}).
\end{DoxyItemize}

It is not recommended to throw \doxylink{structcimg__library_1_1CImgException}{CImg\+Exception} instances by yourself, since they are expected to be thrown only by \textbackslash{}\+CImg. When an error occurs in a library function call, \textbackslash{}\+CImg may display error messages on the screen or on the standard output, depending on the current \textbackslash{}\+CImg exception mode. The \textbackslash{}\+CImg exception mode can be get and set by functions \doxylink{namespacecimg__library_1_1cimg_ac8ac1e757018949dcf8b82495932ba95}{cimg\+::exception\+\_\+mode()} and \doxylink{namespacecimg__library_1_1cimg_aa11ed1e7e4489216ea90a3bc150638df}{cimg\+::exception\+\_\+mode(unsigned int)}.

\end{DoxyParagraph}
