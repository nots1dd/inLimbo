This project uses a {\ttfamily Makefile} to automate the build process. The {\ttfamily Makefile} supports different build configurations to help with development, testing, and production. Below are the various types of builds you can perform and how to use them.

You can build the in\+Limbo in two ways\+:


\begin{DoxyEnumerate}
\item Makefile (uses CMake)
\item Manually building using CMake commands
\end{DoxyEnumerate}\hypertarget{md_BUILD_autotoc_md33}{}\doxysection{\texorpdfstring{Available Build Targets}{Available Build Targets}}\label{md_BUILD_autotoc_md33}
\hypertarget{md_BUILD_autotoc_md34}{}\doxysubsection{\texorpdfstring{1. {\bfseries{all}}}{1. {\bfseries{all}}}}\label{md_BUILD_autotoc_md34}

\item {\bfseries{Description}}\+: This will run the {\ttfamily build-\/all} target, which first executes the initialization script ({\ttfamily init.\+sh}) and then proceeds with the build.

\hypertarget{md_BUILD_autotoc_md60}{}\doxysection{\texorpdfstring{{\bfseries{MANUAL BUILD}}\+:}{{\bfseries{MANUAL BUILD}}\+:}}\label{md_BUILD_autotoc_md60}
\begin{DoxyWarning}{Warning}
~\newline
 Proceed with manual build {\bfseries{ONLY}} if you know what you are doing. ~\newline
~\newline

\end{DoxyWarning}

\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https://github.com/nots1dd/inLimbo.git\ }
\DoxyCodeLine{cd\ inLimbo/}
\DoxyCodeLine{./init.sh\ \#\ to\ initialize\ the\ repository\ with\ some\ important\ headers}

\end{DoxyCode}


Then building commands\+: ({\bfseries{WILL NOT COMPILE IF {\ttfamily init.\+sh} IS NOT RUN FIRST}})


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/S\ .\ -\/B\ build/}
\DoxyCodeLine{cmake\ -\/-\/build\ build/}
\DoxyCodeLine{./build/inLimbo}

\end{DoxyCode}


To build in\+Limbo {\bfseries{GLOBALLY}}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/S\ .\ -\/B\ build\ -\/DGLOBAL\_BUILD=ON\ }
\DoxyCodeLine{cmake\ -\/-\/build\ build/\ }
\DoxyCodeLine{cd\ build\ }
\DoxyCodeLine{sudo\ make\ install\ \#\ will\ put\ the\ binary\ in\ /usr/bin\ and\ respective\ inLimbo.desktop\ and\ logo\ in\ /usr/share}

\end{DoxyCode}
