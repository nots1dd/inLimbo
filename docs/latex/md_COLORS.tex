Color Schema Documentation for in\+Limbo

This document explains the color schema used in \href{https://github.com/nots1dd/inLimbo}{\texttt{ {\bfseries{in\+Limbo}}}}, including all available colors, their respective RGB values, and how to configure them in the {\ttfamily config.\+toml} file.\hypertarget{md_COLORS_autotoc_md62}{}\doxysection{\texorpdfstring{Overview}{Overview}}\label{md_COLORS_autotoc_md62}
The in\+Limbo project uses a set of predefined true colors, each mapped to an {\ttfamily ftxui\+::\+Color\+::\+RGB} value (True Color Palette). This allows for a variety of colors to be used within the application to enhance the user interface.

\begin{DoxyNote}{Note}
~\newline
 {\bfseries{in\+Limbo uses TRUECOLORS}}!! ~\newline
~\newline
 in\+Limbo uses {\ttfamily ftxui\+::\+Color\+::\+RGB} to avoid any terminal emulator color schema messing up the colors ~\newline
~\newline
 For example, if you have {\ttfamily pywal} or {\ttfamily wallust} setup with your terminal, the colors would {\bfseries{NOT}} work as intended with {\ttfamily ftxui\+::\+Color} only ~\newline
~\newline

\end{DoxyNote}
\hypertarget{md_COLORS_autotoc_md63}{}\doxysubsection{\texorpdfstring{How to Set Colors in {\ttfamily config.\+toml}}{How to Set Colors in {\ttfamily config.\+toml}}}\label{md_COLORS_autotoc_md63}
To set a color in the {\ttfamily \$HOME/.config/in\+Limbo/config.\+toml} file, use the corresponding color name defined in the color enum. The color name must be set under the appropriate section in the TOML file.

For example, if you want to change the active window color to {\ttfamily Red}, you would include the following in your {\ttfamily config.\+toml}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[colors]}
\DoxyCodeLine{active\_win\_color\ =\ "{}Red"{}}

\end{DoxyCode}


\begin{DoxyImportant}{Important}
~\newline
 The color names are case-\/sensitive and should match exactly as shown in the available color list below. ~\newline
~\newline

\end{DoxyImportant}

\hypertarget{md_COLORS_autotoc_md65}{}\doxysubsection{\texorpdfstring{How to Use Colors in the in\+Limbo}{How to Use Colors in the in\+Limbo}}\label{md_COLORS_autotoc_md65}
Once the color is defined in the {\ttfamily config.\+toml} file, the {\ttfamily parse\+Colors} function in the code will map the string to the corresponding {\ttfamily \doxylink{namespaceTrueColors_aeafabc145297298741b798e8fdc52a6d}{True\+Colors\+::\+Color}} enum value. The color is then used within the in\+Limbo wherever needed.

For example, setting the {\ttfamily active\+\_\+win\+\_\+color} in the {\ttfamily config.\+toml} like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[colors]}
\DoxyCodeLine{active\_win\_color\ =\ "{}Blue"{}}

\end{DoxyCode}


This will configure the in\+Limbo to use the color {\ttfamily Blue} (RGB(0, 0, 255)) for the active window color.\hypertarget{md_COLORS_autotoc_md66}{}\doxysubsection{\texorpdfstring{Example of {\ttfamily config.\+toml}}{Example of {\ttfamily config.\+toml}}}\label{md_COLORS_autotoc_md66}

\begin{DoxyCode}{0}
\DoxyCodeLine{[colors]}
\DoxyCodeLine{active\_win\_color\ =\ "{}Red"{}}

\end{DoxyCode}


This would set the {\ttfamily active\+\_\+win\+\_\+color} to red (RGB(255, 0, 0)).
